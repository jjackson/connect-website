Readers: 15,000 reading-glasses pairs through Connect's verified-payment model — and we have not run the cohort yet.

Readers is the first non-MNCH program on the Connect catalog. RestoringVision is supplying 15,000 pairs of reading glasses; Connect is supplying the FLW screening, distribution, and verified-payment scaffolding. LLOs are finalized (eHA in Nigeria, C3HD); contract signing is pending. This page is honest about what we know and what we will only know once the first cohort runs.

Q1 · platform fit hypothesis ⟶ insights#readers

Does the Connect verified-payment model port to a non-MNCH commodity-distribution domain?

Connect's evidence base sits in maternal-newborn-and-child health: CHC, KMC, ECD all map cleanly to the platform's verified-visit denominator. Readers is different. The denominator is a fitted pair of reading glasses, dispensed door-to-door to adults with presbyopia. The screening is a quick visual-acuity check; the verification is a photo of the dispensed pair plus the recipient. Whether the same Connect machinery — Trial Run LLO activation, the four-layer verification stack, performance-based payment — produces the same operational reliability in this domain is the empirical question the cohort will answer.

The architectural reasons to expect a yes: the verification stack does not depend on MNCH-specific signals; the LLO economics work the same way; the Trial Run model is platform-portable. The architectural reasons to expect surprises: door-to-door adult distribution is a different routing problem than child-targeted home visits, and the screening protocol (visual-acuity check) is a different competence than weighing a baby. We are deliberately not transferring CHC's 22% cost-reduction curve to Readers in our funder briefings; we will measure and report.

What we expect to learn that we cannot learn from CHC, KMC, or ECD: how the Connect platform behaves when the unit of work is a one-time commodity dispense rather than a recurring health visit. The verification denominator changes — a fitted pair of glasses is the unit, not a per-child encounter — and the LLO operational pattern changes. Door-to-door adult distribution requires different FLW dispatch logic, different routing, and a different screening time-per-encounter. Whether the platform's app prompts, the LLO supervision pattern, and the four-layer verification stack survive that translation cleanly is a question the cohort will answer empirically. We want to be wrong here in ways we can publish, not surprised in ways we cannot.

Source: Connect Readers program design; RestoringVision partnership documents.
Q2 · LLO selection · I1 · N4 triangulated ⟶ deliver#trial-run

How do we select LLOs for a domain where Connect has no prior field data?

The LLO funnel for Readers re-uses the Trial Run model that produced the 65% conversion rate in CHC1. The selection has been narrowed earlier in the funnel because the partner pool for adult vision distribution is smaller than the CHV pool for child-health campaigns: eHA in Nigeria and C3HD have been finalized as the first two LLOs, with contract signing pending.

Both LLOs have prior measured performance from earlier Connect-adjacent work or from RestoringVision's prior distribution rounds. The Trial Run terms still apply: the first cohort is sized for verified-pair-delivery measurement, payment is per-verified-pair, and the conversion gate is on the same data the full program will run on. The departure from CHC is that we are not running a 100-LLO funnel for Readers — the partner ecosystem is smaller, so the screening happens earlier and the contracted partners go directly into a measured first round.

What we will report once the cohort runs: per-LLO verified-pair-delivery rates, screening protocol adherence, dispensation-to-screening-fit ratios (how often the dispensed pair matched the screening prescription), and cost-per-verified-pair. Until then, the page reports what we have committed to — not field outcomes we don't have. This is the v10c knowledge-graph commitment in action: a program with no field data gets a hypothesis badge in the insights index, and its claims on this page are framed as design intent, not measurement. When the cohort closes, the badge changes; until then, it doesn't. The Readers entry is the cleanest place on the site to see how the badging system handles a pre-cohort program — every claim sits behind a hypothesis tag, and the hypothesis tag is itself a methodology commitment to upgrade the badge only when measured data lands.

Source: Trial Run model — FP Final Report (Feb 2026), CHC. Readers-specific LLO selection: program design documents. [1]
Q3 · what the cohort will tell us hypothesis

What questions about the Connect platform will the Readers cohort answer that CHC cannot?

The Readers cohort is structured to answer three questions Connect has not had data on before. First: does the Trial Run conversion rate hold outside CHC? The published 65% figure is CHC-specific; we have not yet seen whether it transfers to a different commodity-distribution domain. Second: does the four-layer verification stack work for adult door-to-door distribution as it does for child-targeted home visits? The biometric and GPS layers should port directly; the photo layer and the algorithmic layer (today calibrated on age/gender/MUAC) need to be re-derived for visual-acuity screening data.

Third: does the cost-inversion curve generalize? CHC showed cost per verified visit falling 22% as the program scaled. Whether the same dynamic applies in Readers — and at what cohort size the curve becomes visible — is something the field will tell us. Until we have data, we report no curve. Pre-cohort claims of cost reduction would defeat the methodology this site is built on.

The honest framing for funders considering Readers: this is a deliberate platform extension into a new domain, with platform-portable mechanisms but not yet platform-portable numbers. We will publish the first cohort's results — including cost-per-verified-pair, Trial Run conversion, and verification-layer performance — as the data lands. The methodology commitment is the same one we hold across the rest of the site: claims with no measured evidence carry a hypothesis badge, and they remain badged that way until field data replaces the design intent. A funder reading this page should know exactly how much of what we say is a measurement and how much is a forecast — that's why the badges exist on every entry, including this one.

Source: Connect Readers program design (April 2026).
